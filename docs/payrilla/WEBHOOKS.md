# Webhooks

# Introduction

You can set up webhooks in the merchant Control Panel to be notified of events that happen in your merchant account. This will allow your app to react to these events without the need to poll for status updates.

Every endpoint registered in the gateway will receive a webhook update for all events. It is your server's responsibility to parse the event type and filter for the events you wish to act upon.

The following documentation contains a list of events we currently support, along with the schemas for each event.

# Setting up webhooks

Follow the following steps to set up webhooks for your account:

1. Set up a server with a publicly accessible HTTPS URL that will be called when an event is triggered.
2. Set up a webhook endpoint in Control Panel in the merchant gateway portal.
3. Enable the endpoint in Control Panel. Endpoints that are not enabled will not receive any events.
4. Test your server to ensure it can handle incoming events.
5. Optional: Make note of the secret token that is generated by the gateway, and use it to verify the integrity of the event payload (see below).

# Handling Events

When your server receives an incoming webhook event, it must respond with a 2xx status code in a timely fashion. Otherwise, the request will time out. In the event of a timeout or error, delivery will be reattempted several times.

Due to retries, a single server may receive the same event multiple times. You can check the event ID to ensure your server has not already processed the event. The ID for each event will be unique, regardless of how many times the event is sent.

# Verifying event data

Each webhook request contains an `X-Signature` header, which can be used to verify that the event payload came from our system and has not been tampered with.

To verify the payload, hash the request body using hash-based message authentication code (HMAC) with the SHA256 algorithm, using the signature key of the endpoint (found in Control Panel in the gateway portal) as the secret. Then, compare the resulting hash to the hash in the `X-Signature` header to verify that the event object has not been tampered with.

Example (NodeJS):

```
import crypto from 'node:crypto';

function doesSignatureMatch(xSignatureHeader, signatureKey, body) {
  const hash = crypto.createHmac('sha256', signatureKey).update(body).digest('hex');
  return hash === xSignatureHeader;
}
```
